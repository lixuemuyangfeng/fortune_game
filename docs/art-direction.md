# 美术与关卡线索原则

## 目标

《暴富幻想所》的热点侦查场景应当像同一个世界里的游戏关卡，而不是 PNG 背景上贴一组风格不一致的 SVG 图标。后续素材生产优先使用统一的位图资产流程：先确定关卡镜头、光照、材质和主题边界，再生成背景与前景线索。

## 视觉生产方式

- 当前方向样张：`docs/art-samples/office-clarity-sample.png`。它只作为风格和清晰度参考，不直接作为最终关卡素材。
- 当前第一关 playable raster 版本：`public/assets/office-level/office-raster-v2.png`。它把办公室背景、五个线索和工位还魂机统一在同一张场景图里，点击层只负责热区和反馈。
- 每个关卡先生成一张统一风格的场景底图，底图负责空间、氛围和叙事密度。
- 关键线索作为独立前景资产生成，和底图保持同一相机角度、光照、材质和色彩。
- 不再把扁平 SVG、小图标、抽象 UI 块直接拼到写实或半写实背景上。
- 推荐方向：半写实 3D / clay-render / 轻微夸张道具，边缘清楚、材质统一、适合移动端识别。
- 正式素材优先输出 PNG/WebP；需要交互层的线索可输出带透明通道的前景资产。

## 线索清晰度规则

- 线索可以藏得不显眼，但不能模棱两可。
- 每个线索必须是可命名的具体物件，例如手机、账单、刮刮乐彩票、香炉、许愿墙、工牌、文件夹。
- 禁止把空白电脑屏幕、抽象卡片、模糊 UI 区域、无主体色块当作线索。
- 屏幕类线索必须有明确内容形态，例如 K 线轮廓、机器人图形、红色亏损标记；不要依赖不可读小字。
- 线索边界应在移动端尺寸下仍可辨认；点击热区可以宽容，但视觉主体必须明确。
- 样张或正式素材中不要出现无意义编号、解释标签或教程文字，除非它们是游戏 UI 的一部分。

## 交互动效原则

具体点击后动效任务和实现方案见 `docs/click-animation-plan.md`。

不要试图让整张 AI 生成图整体动起来。动效应加在可交互前景层或代码叠加层上：

- 选中线索：边缘高亮、轻微呼吸、描边、halo 或 pulse。
- 行情线索：叠加一条发光 K 线，让它像心电图一样短促跳动。
- 人物或情绪道具：可用短 sprite sheet、透明 WebM 或 CSS 粒子表现叹气、抖动、冒烟。
- 默认状态要融入场景；反馈状态由代码控制，避免把发光和提示永久画死在素材里。

## 界面审美与交互规范

这些规则用于保持不同设备、不同编码会话下的第一关体验一致。

- 界面不是运营后台。第一屏必须像游戏处理室，而不是数据看板、任务系统或管理控制台。
- 第一关只展示第一关需要的信息：地点、时间、关卡名、当前目标、线索收集、场景画面、下一关入口。不要提前铺开后续关卡档案。
- 标题层级要克制：`暴富幻想所` 是品牌锚点，`键盘声变轻了` 是本关主标题；不要再叠加“第 1 间处理室”“黄金破防日”等重复标题。
- 地点和时间使用同一套标签样式，保持同一行内视觉语言一致。
- 任务区要一眼可读：目标标题、完成数、回魂进度条、5 个短灯位即可。不要使用英文 `MISSION/CLEAR`，不要做笨重方格槽或后台感步骤条。
- 已找到线索用游戏化证据板、便签、票据、钉子等形式收纳；不要一字排开成运营标签。
- 完成态保持收束：保留场景画面、证据板、下一关按钮和“证据袋已封口”状态。处理线、排行榜、资源产出等系统不要出现在第一关通关页，避免结算页变成后台面板。
- 周启明状态优先反映在右侧大图里，而不是左侧文字卡。进度低时用叹气、暗淡、低能量；找齐后可以用小幅度放松眼线、嘴角弧、轻微回魂光、光点、机器响应。不要使用大面积贴纸或明显补丁式五官。
- 任何提示都不能破坏找茬乐趣。鼠标 hover 不应暴露热区框；只有主动提示、点击命中、已找到状态可以出现可见反馈。
- 移动端必须先检查文字是否完整显示。说明卡和浮层不得被场景边缘裁切，按钮文字不得挤压换行到不可读。
- 动效要短、准、少：点击命中可以有局部脉冲、K 线跳动、纸片弹起、刮奖光泽、聊天戳章；不要让整张 AI 场景图乱动。
- 色彩以深绿处理室、暖黄任务反馈、低饱和纸张/办公室光为主。避免紫蓝渐变、纯科技 SaaS 风、营销落地页风。

## 关卡主题边界

第一关不能变成所有热点的大杂烩。每个关卡只承载自己的核心情绪，跨主题线索只能作为环境里的轻量压力源。

### V0.1 第一关：打工人办公室

核心情绪：工作压力、踏空焦虑、老板压迫、轻度暴富幻想。

适合线索：

- 同事或自己手机里的金价走势。
- 老板消息、会议通知、降本增效文件。
- 基金或账户亏损记录。
- 抽屉里的刮刮乐彩票。
- AI 替代岗位新闻或工具窗口。

不适合塞入第一关：

- 朋友圈收益墙。
- 财神庙香炉和许愿墙。
- 彩票店完整柜台。
- 财经直播间主播台。
- 地铁、出租屋、大厂茶水间等独立场景主体。

### V0.5 MVP 三关

- 办公室：工作场景里的财富焦虑入口。
- 朋友圈：社交比较、收益截图、房贷压力、AI 裁员热帖。
- 财神庙：低成本幻想、求财仪式、彩票摊、宏观玄学、老板来电的荒诞并置。

### 后续关卡池

产品蓝图里的完整候选场景包括：

- 打工人办公室。
- 财经直播间。
- 地铁通勤。
- 出租屋。
- 彩票店。
- 财神庙。
- 大厂茶水间。
- 朋友圈界面。

除 MVP 已选的办公室、朋友圈、财神庙外，其余候选场景应该作为独立关卡扩展，而不是提前挤进办公室第一关。
