# 点击后动效任务与解法

## 目标

第一关下一步要增强“点中线索后”的反馈，但不能回到“鼠标划过就露答案”的状态。隐藏线索默认应保持安静；只有玩家点击、使用提示、或完成线索后，才出现明确反馈。

## 交互原则

- 默认隐藏态：不显示边框、遮罩、闪光或大面积提示。
- 点击命中：只在被点击线索附近出现短促反馈，反馈结束后保留小型已找到标记。
- 提示态：可以短暂显示目标范围，但必须由“给个提示”触发。
- 完成态：机器和线索可以一起响应，但不要遮挡其他未找线索。
- 动效由代码叠加层控制，不把永久发光、编号、说明文字画进底图。

## 第一关动效方案

### 亏损曲线

点击后在显示器曲线区域叠加一条红色折线闪动，像心电图短促抖一下，然后收束成小 `✓`。

### 金价手机

点击后在手机屏幕内叠加一条金色上涨线，从左到右扫过一次，末尾轻微跳动。不要扩大到手机外，也不要在 hover 时出现。

### owner 意识消息

点击后右侧聊天屏幕的目标消息轻微震动一次，叠加一枚小型“已归还”印章感反馈，随后只留小 `✓`。

### 花呗便利贴

点击后便利贴轻轻弹起并落回桌面，同时出现一条短促纸张抖动阴影。

### 刮刮泪

点击后在刮刮乐彩票表面做一次刮痕扫光，随后出现小 `✓`。不要做中奖烟花，避免误导玩法含义。

### 工位还魂机

每找到一个线索，机器波形屏幕短促亮一下。找齐五个后，机器波形变稳定，并吐出结算文案。

## 技术解法

- 在 `SceneHotspot` 上增加可选 `animationKind`，例如 `kline`, `chat`, `paper`, `scratch`, `machinePulse`。
- 在 `renderHotspot` 输出 `data-animation-kind` 和命中后的状态类。
- 用 CSS pseudo-element 或独立 overlay span 绘制局部动效。
- 继续使用当前的内嵌热区模式：底图负责视觉，按钮只负责点击判定和状态反馈。
- 对复杂动效再考虑 sprite sheet 或透明 WebM；第一版优先用 CSS 完成。

## 当前落地状态

- 已在 `SceneHotspot` 上加入 `animationKind`，第一关五个线索均已配置。
- 已用独立 `.hotspot-effect` overlay 承载点击后动效，避免改变命中框和 hover 行为。
- 已为内嵌工位还魂机加入短促波形脉冲，随每次命中触发。
- 当前实现仍是 CSS 第一版，后续可替换为 sprite sheet 或透明 WebM。

## 验收标准

- 未 hover、未提示、未点击时，玩家看不到任何热区框。
- 点击可见物件主体任意合理位置都能触发。
- 动效局限在对应线索附近，不遮挡其他线索。
- 低性能或 `prefers-reduced-motion` 下仍可正常完成关卡。
- `npm run build`, `npm test`, `npm run test:e2e` 通过。
