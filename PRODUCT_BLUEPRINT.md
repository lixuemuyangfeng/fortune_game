# 《暴富幻想所》产品蓝图

## 1. 产品定义

《暴富幻想所》是一款以社会财富热点为内容燃料的轻交互经营小游戏。产品不提供投资建议、不做真实行情预测、不设置押注或现金回报，而是把黄金、AI、币圈、房价、裁员、彩票、降息、财报等热点转译成“财富焦虑梗”，让用户通过找梗、情绪分拣、经营升级、图鉴收集和人格分享消耗碎片时间。

一句话定位：

> 经营一家专门处理人类财富焦虑的幻想机构，每天看看世界又用什么方式提醒你没发财。

核心体验：

- 热点侦查室：在日常场景中找出隐藏的财富焦虑点。
- 情绪分拣室：把热点事件分拣成嘴硬、破防、眼红、幻想、假装懂等情绪资源。
- 暴富幻想所：用资源升级设施、解锁卡牌、收集称号、生成分享图。
- 广告激励：用户主动看广告换取提示、翻倍、加速、额外挑战次数、隐藏卡。

完整关卡叙事线见 `docs/level-campaign.md`；美术和线索生产规则见 `docs/art-direction.md`。

产品目标：

- 用社会热点完成拉新和日更召回。
- 用轻交互完成碎片化时长消耗。
- 用图鉴、经营和限时热点制造留存。
- 用激励广告完成主要变现。
- 同时兼容微信小游戏、抖音小游戏，并预留服务端扩展能力。

## 2. 目标用户

核心用户：

- 22-40 岁泛财经围观用户。
- 对黄金、AI 股、币圈、房价、裁员、彩票、降息等词汇有情绪反应。
- 有财富焦虑、踏空感、暴富幻想，但不一定真的深度投资。
- 习惯短视频、小游戏、抽卡、签到、收菜、分享人格测试。

非目标用户：

- 专业投资者。
- 硬核竞技玩家。
- 未成年人。
- 期待真实收益、荐股、预测工具的人。

核心心理：

- FOMO：是不是又错过了什么。
- 反向安慰：原来大家都没发财。
- 荒诞共鸣：这说的不就是我。
- 差一点完成：再看一次广告就能拿到隐藏卡。
- 身份表达：我要把这个扎心人格发给朋友。

## 3. 商业逻辑

增长链路：

1. 今日热点或分享图触达用户。
2. 用户进入每日热点侦查或情绪分拣。
3. 结算获得财富人格、热点卡、经营资源。
4. 用户进入暴富幻想所升级设施、收集图鉴。
5. 在提示、翻倍、加速、复活、隐藏卡处触发激励广告。
6. 用户分享人格图或挑战结果，带来新用户。
7. 次日热点刷新，召回用户。

收入模型：

> 广告收入 = DAU × 人均广告展示次数 × 广告完成率 × 填充率 × eCPM

主要变现：

- 激励视频广告：核心收入。
- 插屏广告：低频使用，只放在自然关卡结算后。
- 原生广告：后期用于主城空位、卡牌详情页、任务页。
- 轻内购：后期可选，仅售卖装饰、无数值优势道具、月卡免广告等，MVP 不做。

广告设计原则：

- 激励广告必须自愿。
- 广告按钮明确标注。
- 不把广告伪装成系统按钮。
- 不用广告换取押注机会、真实收益、可兑换权益。
- 每个广告都要给即时、明确、可兑现的收益。

## 4. 核心玩法结构

### 4.1 主框架：暴富幻想所

用户经营一家荒诞的财富情绪处理机构。

初始设施：

- 黄金香炉：产出眼红值。
- AI 神坛：产出错过幻觉。
- 彩票垃圾桶：产出暴富幻想。
- 房贷粉碎机：产出破防值。
- 朋友圈截图墙：产出嘴硬值。
- 降息许愿池：产出宏观玄学值。

经营动作：

- 收取离线收益。
- 升级设施。
- 派遣员工处理热点。
- 解锁热点卡图鉴。
- 完成每日订单。
- 生成今日财富人格图。

### 4.2 部门一：热点侦查室

玩法来源：找茬/找梗。

用户进入一个场景，找出隐藏的财富焦虑点。每个点被找出后转化为“热点证据”。

场景示例：

- 打工人办公室。
- 财经直播间。
- 地铁通勤。
- 出租屋。
- 彩票店。
- 财神庙。
- 大厂茶水间。
- 朋友圈界面。

隐藏点示例：

- 同事手机里的金价截图。
- 老板发来的降本增效邮件。
- 朋友晒收益截图。
- 基金亏损页面。
- 彩票未中奖小票。
- AI 替代岗位新闻。
- 房贷账单。
- 财报直播弹窗。

奖励：

- 热点证据。
- 经营资源。
- 今日卡牌。
- 人格标签权重。

广告点：

- 提示一个隐藏点。
- 放大镜 10 秒。
- 延长时间。
- 解锁隐藏热点点位。
- 额外侦查次数。

### 4.3 部门二：情绪分拣室

玩法来源：情绪分类挑战。

用户把热点证据或热词卡快速拖入对应情绪机器。

情绪机器：

- 眼红机。
- 嘴硬机。
- 破防机。
- 幻想机。
- 假装懂机。
- 宏观玄学机。

示例映射：

- 朋友晒收益 -> 破防机。
- 我再等等 -> 嘴硬机。
- 彩票开奖 -> 幻想机。
- 降息会议 -> 假装懂机。
- 黄金新高 -> 眼红机。

奖励：

- 对应情绪资源。
- 连击奖励。
- 设施经验。
- 热点卡碎片。

广告点：

- 自动分拣 10 秒。
- 错误次数清零。
- 延长时间。
- 双倍产出。
- 额外挑战次数。

### 4.4 图鉴与人格系统

图鉴类型：

- 热点卡：黄金、AI、币圈、房价、裁员、彩票等主题。
- 身份卡：精神股东、精准踏空、嘴硬观望者、朋友圈截图受害者。
- 设施卡：黄金香炉、AI 神坛、房贷粉碎机等。
- 事件卡：金价暴击日、AI 封神夜、彩票最后希望日。

人格计算：

- 根据当日找出的热点、分拣选择、失败次数、广告触发点、设施等级加权生成。
- 结果只表达娱乐标签，不表达投资建议。

人格示例：

- 黄金财神门外汉。
- AI 浪潮岸边观众。
- 精准踏空型打工人。
- 朋友圈收益截图受害者。
- 嘴硬观望型穷人。

分享图内容：

- 今日人格。
- 踏空指数。
- 嘴硬指数。
- 今日热点主题。
- 一句扎心签文。
- 小程序码或抖音小游戏入口。

## 5. MVP 版本

### 5.1 MVP 目标

验证三个问题：

- 用户是否愿意玩“财富焦虑找梗 + 情绪分拣”。
- 用户是否愿意为提示、翻倍、隐藏卡看广告。
- 每日人格图是否具备分享潜力。

MVP 不追求复杂经营深度，只验证核心循环。

### 5.2 MVP 范围

必须包含：

- 微信小游戏与抖音小游戏可共用一套核心代码。
- 暴富幻想所主界面。
- 热点侦查室 3 个场景。
- 情绪分拣室 1 套玩法。
- 6 个基础设施。
- 5 类情绪资源。
- 60 张热点卡。
- 50 条签文。
- 12 个财富人格。
- 1 套分享图模板。
- 4 个激励广告点。
- 基础埋点。
- 内容后台最小能力：JSON 配置或轻后台。

暂不包含：

- 内购。
- PVP。
- 复杂排行榜。
- 实时行情。
- 用户生成内容。
- 复杂员工系统。
- 多语言。

### 5.3 MVP 素材生成

素材清单：

- 主城背景 1 张：暴富幻想所。
- 场景图 3 张：办公室、朋友圈、财神庙。
- 隐藏热点点位每场景 8-12 个。
- 设施图标 6 个。
- 情绪机器图标 5 个。
- 热点卡面 60 张。
- 人格徽章 12 个。
- 分享图模板 1 套。
- UI 基础组件：按钮、弹窗、卡片、进度条、资源栏。
- 广告道具图标：提示、放大镜、双倍、自动分拣。

生成方式：

- 第一阶段可用 AI 生成位图草稿，再由美术统一风格。
- 关卡底图、前景线索、动效反馈必须遵守 `docs/art-direction.md`，避免把第一关做成跨场景素材大杂烩。
- 卡牌和图标优先模板化：固定背景、主题色、主体图、短标题。
- 找茬场景必须做点位坐标表，素材和配置分离。
- 所有真实公司 Logo、行情截图、新闻截图避免直接使用，用自制拟物图标和虚构 UI 表达。

素材配置示例：

```json
{
  "sceneId": "office_gold_day",
  "theme": "gold",
  "background": "scene/office_gold_day.png",
  "hotspots": [
    {
      "id": "phone_gold_price",
      "x": 412,
      "y": 238,
      "radius": 36,
      "evidenceId": "gold_price_screenshot",
      "hint": "同事的手机有点刺眼"
    }
  ]
}
```

### 5.4 MVP 方案设计

日常循环：

1. 进入主城，领取离线收益。
2. 可看广告翻倍。
3. 进入热点侦查室，找出 5-8 个热点点位。
4. 获得热点证据。
5. 进入情绪分拣室，把证据拖入情绪机器。
6. 获得资源和热点卡。
7. 升级设施或抽一张今日卡。
8. 生成今日财富人格图。
9. 分享或继续挑战。

MVP 广告点：

- 离线收益翻倍。
- 找茬提示。
- 情绪分拣双倍产出。
- 解锁今日隐藏人格。

MVP 指标：

- 新用户首局完成率 >= 70%。
- D1 留存 >= 25%。
- 人均激励视频次数 >= 1.5。
- 激励广告完成率 >= 80%。
- 分享图生成率 >= 15%。
- 分享点击转化率 >= 5%。
- 单次会话时长 3-6 分钟。

### 5.5 MVP 代码实现

推荐技术栈：

- 引擎：Cocos Creator 3.x 或 LayaAir。
- 语言：TypeScript。
- 包体目标：首包尽量小于平台建议阈值，远程资源按需加载。
- 数据协议：JSON 配置驱动。
- 服务端：Node.js + NestJS 或 Fastify。
- 数据库：PostgreSQL。
- 缓存：Redis。
- 对象存储：COS/OSS/S3 兼容存储。
- CDN：用于图片、音效、配置热更新。

跨平台适配层：

```text
game-core/
  domain/              # 玩法规则、资源计算、图鉴、人格式计算
  config/              # 本地默认配置
  systems/             # 找茬、分拣、经营、广告触发
platform/
  wechat/              # wx API 适配
  douyin/              # tt API 适配
  common/              # 登录、广告、分享、支付预留统一接口
client/
  scenes/              # Cocos/Laya 场景
  ui/
  assets/
server/
  api/
  admin/
  jobs/
  analytics/
```

统一平台接口：

```ts
export interface PlatformAdapter {
  login(): Promise<PlatformLoginResult>;
  showRewardedAd(placement: string): Promise<RewardedAdResult>;
  showInterstitialAd?(placement: string): Promise<void>;
  share(payload: SharePayload): Promise<ShareResult>;
  getLaunchOptions(): LaunchOptions;
  reportEvent(name: string, params: Record<string, unknown>): void;
}
```

配置驱动模块：

- `scene_config`：找茬场景、点位、奖励。
- `sort_config`：情绪分拣题库、正确映射、难度曲线。
- `facility_config`：设施等级、产出、升级消耗。
- `card_config`：卡牌、稀有度、主题、图鉴归属。
- `personality_config`：人格标签、权重、分享文案。
- `ad_placement_config`：广告点、每日上限、奖励倍率。

服务端 MVP 接口：

- `POST /auth/login`：平台登录。
- `GET /config/bootstrap`：启动配置版本。
- `GET /config/:version`：获取配置。
- `GET /user/profile`：用户基础数据。
- `POST /user/progress`：进度上报。
- `POST /game/session/start`：开始一局。
- `POST /game/session/finish`：结算一局。
- `POST /ad/reward/claim`：广告奖励确认。
- `POST /share/report`：分享事件上报。
- `POST /analytics/events`：批量埋点。

MVP 防作弊：

- 关键奖励服务端结算。
- 客户端上传行为摘要，不直接信任最终奖励。
- 广告奖励需要平台回调或客户端完成事件加服务端频控。
- 每日广告次数、挑战次数、资源收益设置上限。
- 配置版本签名，降低篡改配置风险。

## 6. 分期里程碑

### V0.1 原型验证版

周期：2-3 周。

目标：

- 快速验证交互是否成立。
- 不接正式广告。
- 不做完整服务端。

素材生成：

- 1 张主城草图。
- 1 张办公室找茬场景。
- 20 个热点点位和图标。
- 20 张卡牌。
- 10 条人格文案。

方案设计：

- 只做热点侦查室。
- 找出热点点位后直接生成人格。
- 无经营深度，只保留资源展示。

代码实现：

- 单端 H5 或微信开发者工具可运行。
- 本地 JSON 配置。
- 本地存档。
- 基础埋点打印。
- 分享图可本地生成。

验收指标：

- 5 分钟内用户能理解玩法。
- 首局完成率 >= 80%。
- 用户愿意主动保存或转发结果图。

### V0.5 MVP 内测版

周期：4-6 周。

目标：

- 打通找梗、分拣、经营、广告、分享完整闭环。
- 同时跑通微信小游戏和抖音小游戏构建。

素材生成：

- 3 个找茬场景。
- 60 张热点卡。
- 6 个设施。
- 5 台情绪机器。
- 12 个人格徽章。
- 50 条签文。
- 1 套分享图模板。

方案设计：

- 主城 + 热点侦查室 + 情绪分拣室。
- 设施升级 1-10 级。
- 每日任务 5 个。
- 激励广告 4 个点位。
- 每日主题配置。

代码实现：

- 完成平台适配层。
- 完成用户登录、存档、配置拉取。
- 完成广告 SDK 接入。
- 完成分享能力。
- 完成基础服务端。
- 完成埋点上报。

验收指标：

- 微信/抖音双端可审核包。
- 广告奖励可正确发放。
- 配置可热更新。
- D1 留存、广告完成率、分享率可统计。

周期：6-8 周。

目标：

- 支持小规模投放和自然传播。
- 开始运营日更热点。

素材生成：

- 8-10 个找茬场景。
- 150 张热点卡。
- 30 个人格。
- 20 个设施皮肤。
- 100 条签文。
- 7 套分享图主题。
- 节日和财报季模板。

方案设计：

- 每日热点主题。
- 图鉴收集。
- 设施升级 1-30 级。
- 连续回访奖励，避免过强诱导。
- 轻排行榜：只排娱乐指标，如踏空指数、嘴硬值。
- 新手 3 日体验优化。

代码实现：

- 后台内容管理系统。
- 配置发布审核流。
- A/B 测试能力。
- 用户分层广告频控。
- 基础反作弊。
- CDN 资源分包。
- 灰度发布。

验收指标：

- D1 留存 >= 30%。
- D7 留存 >= 10%。
- 人均激励视频 >= 2.5。
- ARPDAU 达到初步投放模型要求。
- 分享带新占新增用户 >= 10%。

目标：

素材生成：

- 轻社交队伍：幻想所联盟。

代码实现：

### V2.0 规模化运营版

周期：10-16 周。

目标：

- 支撑流量上涨和多活动并行。

素材生成：

- 30+ 找茬场景。
- 500+ 热点卡。
- 100+ 人格。

方案设计：

- 用户回流活动。

## 7. 微信小游戏与抖音小游戏兼容方案

### 7.1 客户端策略

采用“一套游戏核心 + 双平台适配”的结构。

共用：

- 游戏规则。
- UI 组件。
- 资源系统。
- 找茬逻辑。
- 分拣逻辑。
- 经营逻辑。
- 配置解析。
- 埋点规范。

平台差异封装：

- 登录。
- 广告。
- 分享。
- 启动参数。
- 文件系统。
- 开放数据。
- 支付预留。
- 性能上报。

### 7.2 平台差异

微信重点：

- 社交分享、群传播、小程序码。
- 好友关系和开放数据域能力。
- 小游戏包体、分包、缓存管理。
- 激励广告稳定性和审核规范。

抖音重点：

- 短视频挂载。
- 直播间互动。
- 内容流转化。
- 抖音账号体系。
- 视频素材传播。

实现原则：

- 不在玩法层直接调用 `wx` 或 `tt`。
- 所有平台能力走 `PlatformAdapter`。
- 分享图生成逻辑共用，入口参数按平台包装。
- 广告位 ID 配置化。
- 审核版本可通过远程配置关闭敏感词和部分广告点。

### 8.2 内容转译规则

真实信息只用于事件背景，不输出判断。

示例：

- 真实事件：黄金价格上涨。
- 游戏表达：黄金香炉今日香火暴涨，眼红值产出 +30%。

- 真实事件：某 AI 公司财报受关注。
- 游戏表达：AI 神坛排队上香，错过幻觉产出 +25%。

禁止表达：

- 买入、卖出、看多、看空。
- 预测明天涨跌。
- 稳赚、内幕、翻倍、财富自由教程。
- 用真实行情结果决定可兑换奖励。

## 11. 合规与审核边界

必须遵守：

- 不提供投资建议。
- 不预测真实标的涨跌。
- 不设置押注。
- 不提供现金、实物、代币或可兑换权益。
- 不伪装广告。
- 不面向未成年人做成瘾诱导。
- 不直接使用真实公司 Logo、未经授权新闻截图、行情截图。
- 明确标注娱乐内容，不构成投资建议。

审核策略：

- 审核包使用更保守文案。
- 远程配置可关闭敏感热点。
- 广告频控可远程调整。
- 分享文案避免“赚钱、暴富教程、内幕、预测”等词。
- 平台素材分别适配微信和抖音审核口径。

## 14. 关键决策

第一阶段不要追求“像羊了个羊一样极难”，而是先验证用户是否接受题材和交互。难度曲线可以后置优化。

第一阶段不要做真实行情，使用人工热点和娱乐化转译，降低合规和数据成本。

第一阶段不要做复杂买量，先靠小流量测试首局完成率、广告完成率、分享图点击率。

第一阶段不要做多玩法合集，所有玩法都要服务“暴富幻想所经营”这一个主框架。

最终目标形态：

> 前台是每日财富焦虑互动挑战，后台是暴富幻想所经营收集，增长靠人格分享图，收入靠自愿激励广告。
